Aprendiendo HTML5

Esta página explica qué es HTML5, cómo se estructura un documento, cuáles son sus etiquetas más importantes y por qué es fundamental en el desarrollo web actual.

Comenzar

¿Qué es HTML5?

HTML5 es la quinta y más reciente versión del lenguaje HTML (HyperText Markup Language), el estándar utilizado para crear y estructurar el contenido de las páginas web. Fue desarrollado en conjunto por la W3C (World Wide Web Consortium) y el WHATWG (Web Hypertext Application Technology Working Group), y se estableció como estándar oficial en octubre de 2014.

HTML no es un lenguaje de programación, sino un lenguaje de marcado. Esto significa que utiliza etiquetas (tags) para indicarle al navegador cómo debe mostrar el contenido: textos, imágenes, videos, tablas, formularios y más. Cada etiqueta tiene un propósito específico y le da significado al contenido que envuelve.

Las principales mejoras que trajo HTML5 frente a versiones anteriores son: soporte nativo para audio y video sin necesidad de plugins externos como Flash, nuevas etiquetas semánticas que dan más significado al código, mejor integración con CSS3 y JavaScript, soporte para almacenamiento local en el navegador y mayor compatibilidad con dispositivos móviles.

Estructura básica de HTML5

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi página</title>
</head>
<body>
  <h1>Hola mundo</h1>
</body>
</html>

Todo documento HTML5 debe comenzar con la declaración <!DOCTYPE html>. Esta línea no es una etiqueta HTML, sino una instrucción que le indica al navegador que el documento está escrito en HTML5. Sin ella, el navegador podría interpretar el código de forma incorrecta.

La etiqueta <html> es el elemento raíz que contiene todo el documento. Se recomienda agregar el atributo lang para indicar el idioma de la página, lo cual también mejora la accesibilidad. Dentro de ella hay dos partes fundamentales: <head> y <body>.

El <head> contiene información que el navegador necesita pero que el usuario no ve directamente: el título de la pestaña, el tipo de codificación de caracteres (UTF-8), los estilos CSS enlazados y los metadatos de la página. El <body> es donde va todo el contenido visible: textos, imágenes, botones, videos, formularios, etc.

Etiquetas principales

h1 - h6

Son las etiquetas de encabezado o título. h1 es el más importante y debe usarse una sola vez por página, generalmente para el título principal. Del h2 al h6, los títulos van disminuyendo en jerarquía e importancia visual.

p

Define un párrafo de texto. El navegador agrega un espacio automático antes y después de cada etiqueta <p>. Es la etiqueta más usada para mostrar texto en una página web.

img

Permite insertar imágenes en la página. Es una etiqueta de cierre automático. El atributo src indica la ruta de la imagen y alt proporciona un texto descriptivo alternativo si la imagen no carga.

a

Es la etiqueta de hipervínculo o enlace. Permite navegar hacia otras páginas, secciones del mismo documento o descargar archivos. El atributo href indica la dirección de destino del enlace.

button

Crea un botón interactivo en la página. Puede usarse dentro de formularios para enviar datos o fuera de ellos para ejecutar funciones de JavaScript cuando el usuario hace clic en él.

ul / ol / li

ul crea listas desordenadas (con viñetas), ol crea listas ordenadas (con números o letras), y li define cada elemento de la lista. Siempre deben usarse juntos: ul o ol como contenedor y li para cada ítem.

HTML5 semántico

Las etiquetas semánticas son aquellas cuyo nombre describe claramente el tipo de contenido que contienen. A diferencia de etiquetas genéricas como <div> o <span>, las etiquetas semánticas comunican su propósito tanto al navegador como a otros desarrolladores que lean el código. Esto mejora la organización, facilita el mantenimiento del código y tiene un impacto positivo en el SEO (posicionamiento en buscadores).

header

Define el encabezado de la página o de una sección. Generalmente contiene el logotipo, el nombre del sitio y el menú de navegación principal. Puede haber varios header dentro de una misma página.

nav

Representa un conjunto de enlaces de navegación principales. Se usa para el menú del sitio que permite al usuario moverse entre secciones o páginas. Los buscadores le dan importancia a su contenido.

main

Indica el contenido principal y único del documento. Solo debe existir un elemento <main> por página. Contiene el tema central del sitio, excluyendo encabezados, pies de página y barras laterales.

section

Define una sección temática dentro del contenido. Se usa para agrupar contenido relacionado, como capítulos, temas o bloques de información. Generalmente incluye un encabezado propio.

article

Representa contenido independiente y reutilizable que podría publicarse por separado: una noticia, una entrada de blog, una tarjeta de producto o un comentario. Debe tener sentido por sí solo fuera del contexto de la página.

footer

Define el pie de página del sitio o de una sección. Suele contener información de contacto, derechos de autor, enlaces legales o redes sociales. Al igual que header, puede aparecer más de una vez.

Formularios en HTML5

Los formularios son uno de los elementos más importantes de HTML5, ya que permiten recopilar datos del usuario para enviarlos a un servidor o procesarlos con JavaScript. Se crean con la etiqueta <form>, la cual actúa como contenedor de todos los campos de entrada.

HTML5 introdujo nuevos tipos de input que facilitan la validación de datos directamente en el navegador, sin necesidad de JavaScript adicional. Algunos de estos tipos son: email (valida formato de correo), number (solo acepta números), date (muestra un selector de fecha), tel (para números telefónicos) y url (valida formato de enlace web). También se puede usar el atributo required para hacer un campo obligatorio.

Multimedia en HTML5

Una de las mejoras más importantes de HTML5 fue la incorporación de soporte nativo para contenido multimedia. Antes de HTML5, para reproducir videos o audio en una página web era necesario instalar plugins de terceros como Adobe Flash, lo que generaba problemas de seguridad, rendimiento y compatibilidad. Con HTML5, esto ya no es necesario gracias a etiquetas específicas para cada tipo de contenido.

Imágenes — <img>

La etiqueta img muestra imágenes en la página. Sus atributos principales son src (ruta del archivo), alt (descripción accesible), width y height (dimensiones). Soporta formatos como JPG, PNG, GIF, SVG y WebP.

Audio — <audio>

La etiqueta audio permite reproducir archivos de sonido directamente en el navegador. Con el atributo controls se muestra un reproductor con botones de play, pausa y volumen. Soporta formatos MP3, OGG y WAV.

Video — <video>

La etiqueta video permite insertar videos sin necesidad de plugins externos. Se pueden añadir controles de reproducción con el atributo controls, y se puede definir una imagen de previsualización con poster. Soporta MP4, WebM y OGG.

Accesibilidad

El atributo alt en las imágenes es esencial para la accesibilidad. Permite que los lectores de pantalla describan el contenido visual a personas con discapacidad visual. También es el texto que se muestra si la imagen no puede cargarse.

Conclusión

HTML5 es el cimiento sobre el que se construye toda la web moderna. Su combinación de etiquetas semánticas, soporte multimedia nativo, nuevos tipos de formulario y mejor integración con CSS3 y JavaScript lo han convertido en el estándar indispensable para cualquier desarrollador web.

A lo largo de esta práctica aprendí que HTML5 va mucho más allá de simplemente mostrar texto en pantalla. Permite estructurar el contenido de forma lógica y significativa, lo que facilita tanto la lectura del código por parte de los desarrolladores como su interpretación por los navegadores y motores de búsqueda. También comprendí la importancia de separar responsabilidades: el contenido corresponde a HTML, el diseño visual a CSS, y la funcionalidad e interactividad a JavaScript. Esta separación hace que los proyectos sean más organizados, fáciles de mantener y escalables.
